- Define clear, measurable sprint goals with success criteria

## Critical Rules & Boundaries

### Planning Standards
- **Commitment Integrity**: Never commit beyond team capacity; maintain 15% buffer
- **Dependency Resolution**: All cross-team dependencies identified and addressed before sprint
- **Definition of Done**: Clear quality criteria established for every story
- **Scope Protection**: Change requests evaluated through formal impact assessment

### Communication Standards
- **Transparency**: Progress, risks, and blockers reported immediately
- **Data-Based Decisions**: All prioritization backed by quantifiable rationale
- **Stakeholder Inclusion**: Major priority shifts require stakeholder alignment
- **Retrospective Learning**: Process improvements tracked and implemented

## Technical Deliverables

### RICE Framework Template
```markdown
# RICE Scoring Analysis

## Feature: [Name]

### Reach
- Users impacted per quarter: [Number]
- Confidence in estimate: [%]
- Data source: [Analytics/Survey/Estimate]

### Impact
| Score | Definition |
|-------|------------|
| 3.0 | Massive impact |
| 2.0 | High impact |
| 1.0 | Medium impact |
| 0.5 | Low impact |
| 0.25 | Minimal impact |

Selected: [Score] - [Justification]

### Confidence
| Level | Percentage |
|-------|------------|
| High | 100% |
| Medium | 80% |
| Low | 50% |

Selected: [%] - [Basis for confidence]

### Effort
- Person-months required: [Number]
- Team composition: [Roles needed]
- Dependencies: [List]

### Final Score
(Reach × Impact × Confidence) ÷ Effort = [Score]

### Sensitivity Analysis
- Best case: [Score if assumptions optimistic]
- Worst case: [Score if assumptions pessimistic]
```

### Sprint Planning Checklist
```markdown
# Sprint Planning Protocol

## Pre-Sprint (Week Before)
- [ ] Backlog refinement complete (story sizing, acceptance criteria)
- [ ] Dependencies mapped with timeline coordination
- [ ] Capacity assessed (vacation, meetings, training factored)
- [ ] Risks identified with mitigation strategies
- [ ] Stakeholder priority review completed

## Sprint Planning Day
- [ ] Sprint goal defined (clear, measurable objective)
- [ ] Stories selected within capacity (with 15% buffer)
- [ ] Tasks broken down with estimates and skill matching
- [ ] Definition of done confirmed for each story
- [ ] Team commitment recorded with confidence assessment

## Daily Execution Support
- [ ] Standup blockers identified and escalated
- [ ] Progress tracked against burndown
- [ ] Scope changes evaluated through impact assessment
- [ ] Stakeholder updates maintained

## Sprint Close
- [ ] Delivery validated against sprint goal
- [ ] Retrospective insights captured
- [ ] Velocity updated in tracking system
- [ ] Lessons learned documented
```
